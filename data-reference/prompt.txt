## https://pmcodingcom-sys.github.io/thaiplus/

จากโครงการ @README.md ช่วยวางแผน ออกแบบ ui/ux สำหรับสร้าง Web Application  รูปแบบ Landing Page เน้นเปิดบนมือถือ สร้าง Repository บน Github (เป็นลิ้งค์ url: เช่น https://naivoi-thai-baanbaan.github.io/ThaiHelpsThaiPlus/ ) โดยให้มีหน้าเพจคล้ายคลึง ตามรูป @data-reference/thai-helps-thai-plus-samplePlate.png (ตัวอย่างอ้างอิง)  จากรูปนี้ ขออธิบายแต่ละส่วนหรือการใช้งาน พร้อมให้ปรับปรุง (จากบนไล่ลงมาตามลำดับ) ดังนี้
- โลโก้  ให้เป็น @data-reference/logo.png 
- ข้อความ "ไทยช่วยไทย  พลัส"
- ข้อควรม อธบาย "ลดภาระ ลดค่าครองชีพ"
- แสดง วัน วันที่ เดือน พ.ศ.  ปัจจุบัน (แบบไทย)  และให้เพิ่มบอกเวลา ณ ตอนนี้ รูปแบบ hh:nn:ss ( hh = 0 - 24  ) พร้อมให้ เวลาเดินตลอด (นาฬิกา)
- ส่วนแสดง จำนวนเงินที่เหลือของรัฐบาลสนับสนุน (ผลการแสดงจาก คีย์เลขลงช่อง จำนวเงินที่ต้องการซื้อ --> คลิกคำนวณ --> คลิกบันทึก) มี 2 ส่วน = วันนี้เหลือ จาก 200 บาท , เดือนนี้เหลือ จาก 1000 บาท
- ช่องฟอร์มกรอกจำนวนเงินที่ต้องการซื้อ 
- ปุ่มคำนวณ = เมื่อคลิก จะนำเลขจำนวนเงินที่ต้องการซื้อไปคำนวณ แล้วแสดงผล list down ตามรูป @Calculate.png 
- ปุ่มบันทึก = เมื่อคลิก  เก็บข้อมูลจาก การคำนวณ ไปลงรายละเอียด ที่ส่วน ประวัติการใช้จ่าย (วันที่:เวลา, ยอดซื้อ, รัฐช่วย, คุณจ่าย) ตามรูป @save.png

เมื่อมีผู้ใช้เปิดลิ้งค์ เช่น https://naivoi-thai-baanbaan.github.io/ThaiHelpsThaiPlus/  (ไม่มีการ login) ผู้ใช้งานทำการคำนวณ แล้วบันทึกเสร็จ  ปิดเว็บไป  เวลาหลังจากนั้น ผู้ใช้งานคนเดิมมีคลิกลิ้งค์จะใช้งานต่อ ค่าที่คำนวณคงเหลือ และประวัติการใช้จ่าย ยังคงจำมาแสดงผลเหมือนเดิม

จากที่กล่าวมา : อยากให้บันทึกข้อมูลได้ด้วย มีแนวทางที่เหล่านักพัฒนาป๊อปปูลาร์ใช้กันอย่างไร
